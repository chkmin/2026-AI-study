### 오늘의 키워드
훈련세트와 테스트 세트, 샘플링 편향, 데이터 전처리

**1. 훈련세트와 테스트 세트**
- 문제점: 데이터를 줘 훈련시키고 같은 데이터로 평가하여 정확도가 100%가 나오는 것 => 따라서 훈련에 활용하는 데이터와 평가에 활용하는 데이터 구분 필요!!

- 해결 방안 : 
1. 평가를 위한 데이터를 새로 준비
2. 준비된 데이터 중 일부를 떼어 평가에 사용 => 2번 사용!!!

**2. 샘플링 편향**
- 문제 발생: 전체 데이터 중 처음 35개 데이터와 나머지 14개 데이터를 분리하여 학습시켰더니 score() 매서드를 통한 평가 결과가 0.0이라는 최악의 결과 발생 => 데이터 분리하는 과정에서 샘플이 섞이지 않고 한쪽으로 치우쳐저 샘플링 편향 발생!

- 해결 방안:
테스트 세트와 훈련 세트에 도미와 빙어 데이터가 골고루 섞이도록 해야함 -> train_test_split() 함수를 통해 비율 조정하여 랜덤하게 섞기

**3.데이터 전처리**
- 문제점: 스케일이 달라 잘못 예측한 값이 생김 => 데이터 전처리 방법 중 표준점수를 활용해 해결

-표준 점수 = (데이터 - 평균) / 표준편차

### 느낀점
평소 다루던 미적분이나 통계학에 비하면 표준 점수라는 통계 공식 자체는 아주 친숙하게 다가왔다. 하지만 이론을 아는 것과 그것을 파이썬 코드로 직접 구현해 스케일을 맞추는 것은 또 다른 문제였다... 코드를 짜면서 실행은 되지만 자잘한 컴파일 에러들 때문에 살짝 애를 먹기도 했지만, 결국 끈질기게 깊게 생각하여 데이터 전처리를 완벽하게 해내고 산점도 그리기까지 완벽하게 해내어 모델이 제대로 돌아가게 만들었을 때 희열감이 들었다.
